Intégration et déploiement continuLe code était mis en commun via un repo sur github. Des revu de code était faites et des tests était fait lors de la mise en commun sur la branche principal.

Project applicatif pour le CHU de Rouen

Reprise d'un application utilisant Symfony (PHP) en back et Angular (JS) en front pour le CHU de Rouen, le but est de faire évoluer leur application avec l'évolution de leur besoin

CompétenceContexte
Participer à une démarche agile et innovanteUtilisation d'un methode Agile Kanban pour l'organisation du projet avec le CHU. Des réunions avec le client était organisé toute les une a deux semaines.
Intégration et déploiement continuLe code était mis en commun via un repo sur gitlab dont je suis en charge. Chaque developement était fait sur des branches indépendantes et des revus de code était organisé avant de merge du code sur la branche principal. Tout dépôt de code sur le repo déclanchais des piplines (Test automatique du code) dans les respect des regles de DevOps.

Lectures suplémentaire

Aimant apprendre et developé mes compétence, je lit régulirement des livres de reference sur le developement et les methodes informatique afin d'approfondire mes compétences. C'est connaissance sont généralement asser ubiquitaire pour pouvoir etre appliquer à l'emsemble des projet où je participe.

CompétenceContexte
Intégration et déploiement continu
  • "DevOps - Intégrez et déployez en continu (avec exemple clés en main)" de Ernesto Leite est un livre qui permet de voir et d'appliquer l'ensemble des principe du DevOps en fesant un projet depuis sont début. Il a également des chapitre sur les méthodes agile du a leur lien étroit et besoin en CI/CD
Intégration et déploiement continu
  • "Clean Code" de Robert Martin. Est un livre parlant de la qualité du code et du profesionalisme chez les developeurs.
  • "The Pragmatic Programmer" de David Thomas. Est un emsemble de conseils d'un developeurs experimenté et reconnu afin de mieux penser son code et en améliorée la qualité.
  • "Design Patterns" du gang des quatre (Erich Gamma, Richard Helm, Ralph Johnson et John Vlissides). Est un livre traitant des patrons de conception. Une maniere d'ecrire son code qui permet de le rendre plus lisible, maintenable et evolutif
  • "The Art of Computer Programming" de Donald Knutt. Est une reference sur algorithmie, une dicipline clé pour crée un code simple et efficace.
